En notes de chiffrage

Quelques questions.

Souverain · trois façons de faire

API européenne, on-prem cloud, ou local.

Trois niveaux de souveraineté, du plus simple au plus intégré :

1 · API souveraine (ex. Mistral)
Les appels passent par un fournisseur européen — données hébergées et traitées dans l'UE, hors juridiction extraterritoriale. Aucune infra à gérer : le client paie sa consommation à Mistral, je ne facture que le setup et le run. Le premier pas souverain, sans friction.
2 · On-prem cloud
Le client loue lui-même un serveur/instance GPU chez un hébergeur souverain (OVH, Scaleway) et le paie en direct ; on y déploie des modèles open weight. Je ne facture que le setup et le run (le service), jamais l'infra.
3 · Local (Box IA)
Machine physique, ma propriété, posée dans les locaux du client — c'est sa raison d'être : la donnée ne quitte jamais les murs. Un loyer tout compris (matériel + service), aucun capital ni facture d'hébergeur côté client. Colocation datacenter possible dans un second temps ; à ce stade, l'on-prem cloud est souvent plus simple.

Les trois gardent la donnée hors des API non européennes. Facturation : API souveraine et on-prem cloud = infra/conso payée en direct par le client + mon service ; local = une seule facture, la mienne.

Honnêtement : sur de très gros volumes par requête, la consommation peut coûter plus cher qu'un forfait — c'est exactement pourquoi l'audit puis le FinOps dé-risquent la bascule avant tout engagement.
